
# Objectifs

* S'adapter aux changements de choix technologiques
* Ajouter des fonctionnalités métier nécessite peu de code une fois le coeur logiciel développé
* Lorsque les objectifs fonctionnels changent, beaucoup de code peut être recyclé
* Peu de bugs atteignent la production
* Il est facile de travailler de façon collaborative

# Propriétés

* Les différents éléments fonctionnels et techniques du code sont isolés les uns des autres 
* Le couplage est faible (non mesuré par sonar)
